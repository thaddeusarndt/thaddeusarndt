## Selected project directions

These are the kinds of systems I am packaging into public, sanitized GitHub repos:

### AI Agent Workspace Lab

A clean-room overview of a multi-device agent workspace: local agents, browser handoff, scheduled jobs, tool orchestration, and approval boundaries. The public version focuses on architecture, mock configs, diagrams, prompt patterns, and safety practices rather than private credentials or personal data.

### Approval-Gated Automation

A case-study style repo showing how to build automations that research, draft, prepare, and report while stopping before sensitive actions like submitting applications, sending messages, changing accounts, spending money, or publishing live changes.

### Research Intelligence and Automation Systems

Public-safe patterns from content-gap discovery, SEO research, prediction-market monitoring, memecoin narrative scanning, and other public-data workflows: collection, filtering, signal tracking, risk controls, dashboards, dry-run execution, and structured reporting. Public examples exclude private strategy logic, live execution paths, account details, credentials, and production-only configuration.

### Creator Commerce and Website Systems

Static and lightweight web systems for creative businesses: portfolio sites, booking funnels, SEO pages, payment flows, and a homemade beat marketplace integration that replaced a third-party subscription with a direct site and Stripe workflow.

### Creative AI, Audio, and Content Tools

Experiments and utilities around music production, AI-assisted content systems, audio workflows, artist operations, and creative automation for publishing, marketing, and media reuse.

### ScenePartner and Creative Practice Apps

A theater scene-practice app concept turned into a working product path: script/scene workflows, practice tooling, mobile release preparation, and creative software designed around real performer needs.

## Principles

- Build useful tools, not demos for demo's sake.
- Turn ambiguous goals into scoped prompts, context packs, testable tasks, and review loops.
- Keep humans in control for sensitive decisions.
- Separate public artifacts from private data, credentials, logs, and personal history.
- Prefer clear documentation, reproducible examples, and safe defaults.
- Treat security and redaction as part of the product, not an afterthought.
